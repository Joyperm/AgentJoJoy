# Gap Reports

Local redacted gap reports generated by the Automated Gap Reporter.

Gap reports are meant to capture reusable workflow friction without
leaking project details. A report must not contain:

- real project names
- remote URLs
- absolute personal or company paths
- credentials, tokens, usernames, IP addresses, or emails
- source code, schema details, proprietary identifiers, or business
  logic

## Files

Use this shape:

```text
gap-YYYYMMDD-HHMMSS.md
```

Collector outputs are generated under:

```text
_collector/
_exports/
```

These generated outputs are local-only and should not be copied into
team repos.
